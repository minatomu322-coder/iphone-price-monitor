# 最終目標「URLを貼るだけ」への設計ロードマップ

最終目標: 商品URLを貼るだけで、相場取得 → 利益計算 → 一次判定 → ChatGPTレビュー用データ生成 →
出品文作成 → システム保存 → 売却後の学習 まで自動で完結する。

## 現在地（実装済み）

```
URL貼り付け（クイック登録）
  └→ 解析プロンプト自動生成 → ChatGPTへ手動コピー（画像・ページ本文を添付）
       └→ ChatGPTが商品情報JSON（相場込み）を生成 → 貼り付けて取り込み
            └→ 商品登録・相場保存・一次判定 まで自動
                 └→ 仕入れ判断用データ出力（ChatGPT最終レビュー）
                      └→ 出品作成用データ出力 → 出品用JSON取り込み（下書き）
                           └→ 本出品（ユーザー手動）
                                └→ 売却記録 → スコアリング・リピート判定・特徴学習（insights出力）
```

人手が必要なのは「ChatGPTとの間のコピー往復」と「最終決定・実操作」だけの状態になっている。
残る自動化対象はコピー往復の部分であり、それはOpenAI API連携（フェーズ2）で置き換え可能。

## フェーズ1（済）: 半自動パイプライン

- クイック登録UI（URL→解析プロンプト生成→商品情報JSON取り込み→一次判定）
- 出品用JSON取り込み（下書き）
- 売却後の学習データ蓄積（スコア・リピート判定・insights出力）

## フェーズ2: OpenAI API連携（コピー往復の自動化）

システム内から `mercari_config.yaml` の `openai:` 設定に従ってAPIを呼ぶ。

- 実装場所: `mercari/gpt_client.py`（新設）に集約し、既存モジュールへは注入する形にする
- 呼び出し単位は既存の手動フローと同じ粒度にする（商品解析 / 仕入れレビュー / 出品文 / 売れ残り分析 / 月次）
  → プロンプトは exports.py の PREAMBLES と docs/chatgpt_prompts.md を共通利用
- 応答は既存のJSONスキーマ（mercari_item_info / mercari_listing_draft）をそのまま使う
  → API化しても取り込み経路は importer.py の1本のままで、手動コピーへいつでも戻せる
- 必須ガード（config済み・実装時に厳守）:
  - APIキーは環境変数のみ（コード・設定へ直書きしない）
  - 1回あたりのコスト上限 / 1日あたりの回数上限
  - 送信前にユーザーが内容を確認できる（confirm_before_send）
  - 利用料金を api_usage テーブルへ記録
  - 失敗時は手動コピー運用へフォールバック

## フェーズ3: 相場の自動取得について（重要な注意）

メルカリには公開APIがなく、サイトの自動スクレイピングは利用規約違反となり
アカウント停止やIPブロックのリスクがある。**メルカリ本体の自動スクレイピングは実装しない。**

代替の選択肢（リスクの小さい順）:
1. ChatGPTのWeb検索能力に相場調査を任せる（フェーズ2のAPI連携でそのまま自動化される）※現行方式
2. 規約上問題のない相場情報源（公式に許可されたAPI・データ提供サービス）が利用可能になった場合のみ、
   `market_snapshots` への自動書き込みアダプタを追加する（`insert_market_snapshot` が受け口）
3. ユーザー自身のブラウザ操作を補助するブックマークレット等で「見ている検索結果を手元で構造化して貼る」方式

いずれの場合も保存先・判定ロジックは現行のまま使えるように、相場の入口は
`MercariDatabase.insert_market_snapshot` に一本化してある。

## フェーズ4: 学習の高度化

- insights出力（済）をChatGPTに定期投入し、返ってきた「仕入れ基準」を
  `mercari_config.yaml` のしきい値へ反映する運用ループを作る
- 蓄積が増えたら（目安100件超）、価格帯×カテゴリー×仕入れ先のクロス集計や
  売れ残り理由と改善結果の相関分析を追加する

## 常に守ること

- 購入・本出品・削除・キャンセル・購入者への送信は自動化しない（ユーザーの最終操作）
- 既存機能を壊さない: スキーマ変更は「列追加のみ」のマイグレーション（`_migrate`）で行う
- 各フェーズでテストを追加し、`python -m unittest tests.test_mercari` を通してからコミットする
