# ChatGPT用プロンプト集

ChatGPT側に貼り付けて使う定型プロンプトです。
システムの各出力（仕入れ判断用・出品作成用・売れ残り分析用・売上分析用）には
依頼文が先頭に付いているのでそのまま貼れますが、より細かく指示したい場合に使ってください。

---

## 1. 商品画像の解析（最初に商品を特定してもらう）

```
この商品画像を解析してください。以下を確認して回答してください。

1. 商品名の推定
2. 型番の推定
3. JANコード候補
4. （カードの場合）カード名・カード番号・レアリティ
5. ブランドの推定
6. 色・容量・サイズ・仕様
7. 写っている付属品
8. 傷・汚れ・白かけ・へこみ・破れなどの状態
9. 商品を特定するために不足している情報・追加で撮るべき写真
10. 偽物や型番違いの可能性があれば警告

それぞれに確信度（高：画像からほぼ特定できる／中：候補は絞れるが追加確認が必要／低：画像だけでは判断できない）を付けてください。
画像から確認できないことは「確認できない」と書いてください。
```

---

## 2. 出品文の作成（システムの「出品作成用データ」と一緒に送る）

```
上記のデータと添付画像から、メルカリ出品用の内容を作成してください。

必要なもの：
- 検索されやすいタイトル（40文字以内）
- 自然で読みやすい説明文（状態・付属品・発送・注意事項を含む。テンプレ感を出しすぎない）
- カテゴリー候補・ブランド候補・商品の状態候補
- 価格3案（早売り価格／相場価格／強気価格）とそれぞれの理由

守ること：
- 画像またはこちらが伝えた情報で確認できない内容は書かない
- 「美品」「未使用」「正規品」は確認できた場合のみ使う
- 欠点を隠さない

最後に、システム取り込み用に以下の形式のJSONを1つ出力してください（コードブロックで）：
{
  "type": "mercari_listing_draft",
  "item_id": <データ内の商品ID。不明なら省略しname を入れる>,
  "title": "...",
  "description": "...",
  "price": <おすすめの価格（数値）>,
  "category": "...",
  "brand": "...",
  "condition": "...",
  "min_price": <最低販売価格（数値）>,
  "shipping_method": "...",
  "shipping_days": "..."
}
```

---

## 3. 仕入れ判断の最終レビュー（システムの「仕入れ判断用データ」と一緒に送る）

システムの出力に依頼文が含まれています。追加で厳しめに見てほしい場合：

```
上記の仕入れ候補を厳しめにレビューしてください。特に以下を疑ってください。
- 相場の商品と本当に同一か（型番違い・セット数違い・付属品差・状態差）
- 売り切れ価格に外れ値が混ざっていないか
- 送料・手数料の見落とし
- 偽物リスク、価格下落リスク、回転率、資金拘束の長さ

結論は「買い／条件付きで買い（条件を明記）／見送り／追加確認が必要（確認事項を明記）」のいずれかで、理由を必ず付けてください。
```

---

## 4. 売れ残り改善（システムの「売れ残り分析用データ」と一緒に送る）

```
上記の売れ残り出品について、値下げ提案の前に「なぜ売れていないか」を
閲覧数・いいね数・価格と相場の差・タイトル・説明文・写真の観点から分析してください。

そのうえで以下を提案してください：
- タイトル改善案（そのまま使える形で）
- 説明文の改善点
- 1枚目写真の改善・追加撮影すべき写真
- 価格変更（するなら金額と理由）／セット販売・バラ売り／他販路
- カテゴリー変更・追加すべき検索キーワード
- 損切りすべきか保有継続か（最低販売価格と資金効率を考慮して）
```

---

## 5. 購入者・出品者対応文の作成

```
メルカリで以下のメッセージが届きました。返信文を作成してください。

条件：
- こちらの立場を守りつつ、感情的にならない
- 証拠として残っても問題ない丁寧な文面
- 値下げ交渉なら、最低販売価格 <金額> 円を下回らない範囲で対応

届いたメッセージ：
<ここに貼る>

状況の補足：
<商品名・現在価格・在庫状況など>
```

---

## 6. 月次分析（システムの「売上分析用データ」と一緒に送る）

システムの出力に依頼文が含まれています。追加で聞きたい場合：

```
上記の実績から、来月の行動計画を作ってください。
- 仕入れを増やすカテゴリー／減らす・撤退するカテゴリー（理由付き）
- 仕入れ先ごとの評価（続ける・条件付き・やめる）
- 資金効率を上げるために今すぐ処分すべき在庫
- 価格設定・値下げ運用の改善点
- もしシステム（記録・通知・画面）側に改善してほしい点があれば、
  Claude Codeへの指示文の形で書いてください
```
